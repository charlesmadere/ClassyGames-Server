Classy Games Server Documentation

This is an attempt to document exactly how to make API calls against the Classy Games server code. All of the below endpoints require HTTP POST requests in order for them to properly gather your given data. All of these endpoints will respond to your input with a UTF-8 encoded JSON string (application/json; charset=UTF-8) as output.

Knowledge of parsing JSON data will be needed to understand how to manipulate the server's output.

1. Example

This example is here so that you can more easily understand the meaning of all of the actual Classy Games Server API endpoints.

Input

This section lists the inputs that any given API endpoint requires in order for the server to properly respond to your request. So in this example's case the list below requires id and board data to be sent via HTTP POST. If you fail to send HTTP POST data to both of those inputs then the server will reject your response with a message notifying you that you did not send the proper amount of data. If you properly send all necessary data then the server may still encounter some issue; in this case the server will notify you of the issue. If you properly send all necessary data and the server does not encounter any issues then you will receive a JSON response containing the data that you requested.

  1. id

    A string that closely resembles an SHA-256 hash.

    example
    Hello, World!
  2. board

    A JSON string that represents the game board.

    example
    Hello, World!

Output

Hello, World!
