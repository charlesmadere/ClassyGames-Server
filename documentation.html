Classy Games Server Documentation

This is an attempt to document exactly how to make API calls against the Classy Games server code. All of the below endpoints require HTTP POST requests in order for them to properly gather your given data. All of these endpoints will respond to your input with a UTF-8 encoded JSON string (application/json; charset=UTF-8) as output.

Knowledge of parsing JSON data will be needed in order to understand how to read and manipulate the server's output.

This document was last updated on BLAH BLAH and is seeing heavy construction as of this time.

1. Example

This example is here so that you can more easily understand the meaning of all of the actual Classy Games Server API endpoints.

Input

This section lists the inputs that the API endpoint requires in order for the server to properly respond to your request. So in this example's case, the list below shows that this endpoint requires id and board data to be sent via. If you fail to send HTTP POST data to both of those inputs then the server will reject your response with a message notifying you that you did not send the proper amount of data. If you properly send all necessary data then the server may still encounter some issue; in this case the server will notify you of the issue. If you properly send all necessary data and the server does not encounter any issues then you will receive a JSON response containing the data that you requested.

  1. id

    This area describes the data for the id parameter. The grey area below this shows an example of what this type of data should look like.

    A3F3B92D38BE145BCBD20911B2AA8E97
  2. board

    This section would detail the data for the board parameter.

    Hello, World!

Correct Output

Here you'll see the exact output you'd receive if you sent the server the above information. But uh, well, that's actually not completely true. The example input data is going to be largely made up, and as such if the server wants an ID, and you send it the ID that is shown in the example, that ID may not exist, which then means that you will not receive the data as shown in this example output. If you do everything right, as specified in the input section above, you will receive data that is identical to this, except that it, well, doesn't contain the same information. This section is mostly useful for writing your parsing code, as you'll know how to read through the received JSON data.

Hello, World!

Alternate Outputs

This area talks about the type of information you'd receive if something is incorrect regarding your data, if the server can't find the information you've requested, if the server encountered an error, or a bunch of other things. Each specific error is further detailed in each API endpoint's documentation.

Some errors are common throughout all of the API endpoints. Please pardon the redundancy!

  1. Malformed Input Data

    Something was wrong with the data that you sent the server.

2. ForfeitGame

Allows a user to forfeit a game. This action can be completed on a game that is either this user's turn or their opponent's turn. After a game has been forfeitted, a push notification is sent out to both users.

Input

  1. user_challenged

    The ID of the current user's opponent. This ID should be obtained from Facebook and is a long.

    9223372036854775807
  2. name

    The name of the current user's opponent. This should be taken from Facebook and is a String.

    Charles Madere
  3. user_creator

    The current user's Facebook ID. As with all of the above inputs, this one should also be gathered from Facebook. This is a String.

    372036854775808
  4. game_id

    The ID of the game that is being forfeitted. This is a String that closely resembles an SHA-256 hash.

Correct Output

POST data received.

Alternate Outputs

  1. No Input Data Detected

    The server did not receive any HTTP POST data. You may have tried to use HTTP GET instead (which the server does not support).

    No POST data detected.
  2. Empty or Incomplete Input Data

    Something was wrong with the data that you sent the server.

    POST data is empty or incomplete.
  3. Malformed Input Data

    Something was wrong with the data that you sent the server.

    POST data is malformed.
  4. Google Cloud Message Failed to Send

    A push notification was attempted but failed. This probably means that you did not do anything wrong; most likely the server is configured improperly. Notify the server administrator, yeah?

    Google Cloud Message failed to send.
  5. Database Connection was Unable to be Established

    A connection to the server database was attempted but failed. Again, you probably didn't do anything wrong. Tell the server administrator!

    Database connection was unable to be established.
  6. Generic Error

    The dreaded meaningless error. You should notify the server administrator if you get this.

    POST data received but an error occurred.
  7. Game is Already Over

    You tried to forfeit a game that was already over. This could potentially happen if the user was viewing an old games list as they maybe hadn't refreshed it in a while.

    Attempted to add a new move to a game that has already been completed!
  8. Could not Find Game with Specified ID

    The game you tried to forfeit? Well apparently it doesn't exist. This is definitely a client side error (unless somehow the entire database has been wiped or something). Check your code that grabs and sends the game's ID.

    Game could not be found with specified ID.
